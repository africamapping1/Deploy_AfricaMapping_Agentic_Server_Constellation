# Backend Socket Destinations

## Purpose

This layer defines the main backend destinations to which Service Availability Matrix subsections may connect.

It exists so socketing has named landing points in the constellation.

This is the backend-destinations layer of the Service Availability Matrix.

## Why This Exists

A socket is only useful if the platform knows where it is meant to land.

The matrix should connect to real operating areas, not abstract future ideas only.

## Main Backend Destinations

### 1. server-07-ai-orchestrator
Strongest use:
- governance interpretation
- review packaging
- approval logic
- escalation logic
- promotion logic

This is the strongest home for:
- readiness sockets
- promotion sockets
- exception sockets

### 2. server-06-monitoring
Strongest use:
- visibility
- drift signals
- maturity summaries
- review freshness
- parity observation

This is the strongest home for:
- parity sockets
- summary sockets
- doctor-style visibility later

### 3. server-08-ai-worker
Strongest use:
- drafting
- comparison
- review assistance
- hygiene support
- low-risk maintenance behavior later

This is the strongest home for:
- agent-assisted sockets
- maintenance sockets
- low-risk enforcement candidates later

### 4. Provisioning and onboarding paths
Strongest use:
- baseline scaffolding
- profile-aware setup
- default file and folder generation
- early registration shaping

This is the strongest home for:
- baseline sockets
- profile sockets
- minimum-presence sockets

### 5. Human operational routines
Strongest use:
- checklist execution
- review cadence
- approvals
- exceptions
- judgment

This is the strongest home for:
- manual review sockets
- early governance sockets
- human signoff sockets

## Important Note

A single matrix subsection may land in more than one destination.

That is normal.

The goal is not one socket per file.

The goal is meaningful operational connection.

## Final Principle

Socketing becomes real when destinations are named clearly.

This layer exists so matrix subsections have real places to land.
