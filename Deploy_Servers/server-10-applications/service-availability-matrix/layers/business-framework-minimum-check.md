# Business Framework Minimum Check

## Purpose

This layer defines how the Service Availability Matrix should check whether the business framework satisfies minimum expected business-definition presence.

It exists so the platform can distinguish between nominal framework presence and meaningful framework readiness.

This is the minimum-check layer for business-framework evaluation.

## Why This Exists

A business framework may appear present because folders and README files exist.

That is not the same as meaningful control-readiness.

The matrix needs a basic minimum check.

## Minimum Check Questions

The matrix should be able to ask:

### 1. Minimum Business Definition Present
Is the minimum business definition explicitly defined?

### 2. Completeness Thinking Present
Is business completeness described in a usable way?

### 3. Readiness Thinking Present
Is business readiness described in a usable way?

### 4. Comparison Basis Present
Is there a stated basis for fair business comparison?

### 5. Profile Expectations Present
Is there a profile-aware interpretation of business strength?

### 6. Summary Discipline Present
Is there a stated discipline for summary trust and upkeep?

## Minimum Check Outcome Types

### Missing
The business-framework control surface is not yet meaningfully present.

### Partial
Some core governance pieces exist, but the framework is not yet strongly evaluable.

### Present
The minimum governance/control surface exists and can support further matrix use.

### Strong
The minimum surface exists and is clearly organized enough for repeatable review.

## Important Note

A minimum check is not a full maturity judgment.

It is the first threshold.

## Final Principle

Before the matrix judges depth, it should confirm presence.

This layer exists so evaluation begins from a stable threshold.
