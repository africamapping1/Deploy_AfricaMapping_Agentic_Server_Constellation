# Run Output Format

## Purpose

This file defines the normal output format of a completed run.

It exists so run results stay reusable and comparable.

## Output Structure

### Capability
- Name:
- Role in system:

### Business Mapping Summary
- Value proposition:
- Customer segments:
- Relationships:
- Channels:
- Activities:
- Resources:
- Partnerships:
- Cost structure:
- Value logic:

### Readiness
- Defined / Partial / Unclear

### Notes
- Risks:
- Unknowns:
- Follow-up needs:

## Why This Matters

A run is only useful if it leaves behind a reusable result.

## Final Principle

Runs should produce outputs that the system can reuse later.
