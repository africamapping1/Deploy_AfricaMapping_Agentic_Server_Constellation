# Run Learning Integration

## Purpose

This file explains how the platform should learn from completed runs.

It exists so every run produces not only decisions, but reusable knowledge.

## Learning Rule

After a run completes, the platform should ask:

- what was clarified
- what repeated pattern appeared
- what can be reused later
- what should be stored in framework memory

## Where Learning Should Go

Reusable learning may be stored in:

- business-framework
- future gis-framework
- tenant summaries
- standards and templates

## Examples

### From imagery acquisition run
Reusable outputs may include:
- imagery source patterns
- quality expectations
- cost considerations

### From visualization run
Reusable outputs may include:
- delivery surface patterns
- user visibility expectations
- display constraints

## Final Principle

A run should leave knowledge behind, not only completion status.
