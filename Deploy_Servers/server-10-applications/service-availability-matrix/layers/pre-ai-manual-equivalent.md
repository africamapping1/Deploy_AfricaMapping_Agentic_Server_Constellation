# Pre-AI Manual Equivalent

## Purpose

This layer explains what the Service Availability Matrix effort would typically have looked like before AI-assisted drafting and structuring.

It exists so the platform can understand the human labor equivalent of the current matrix build.

This is the manual-equivalent layer of the Service Availability Matrix.

## Why This Exists

The matrix now contains significant work in:

- naming
- classification
- comparison
- readiness logic
- parity logic
- profile logic
- onboarding logic
- governance interpretation
- summary maintenance

Before AI, this kind of work would still have been done.

It would simply have required more manual effort and more coordination between roles.

## Typical Pre-AI Roles

A manual build of this kind would often involve some combination of:

- business analyst
- systems analyst
- technical writer
- product or operations lead
- architect or subject-matter reviewer

A small company might compress several of these into one person.

A larger company might spread them across several people.

## Typical Manual Activities

Before AI, work like this would usually require people to manually:

- inspect directory structures
- define naming rules
- compare tenants
- draft policy language
- draft maturity definitions
- maintain summary documents
- track changes across many files
- decide what is comparable and what is not
- rewrite documents repeatedly as the model improves

## Main Manual Challenge

The hardest part was not only writing.

The hardest part was preserving consistency across many related concepts over time.

That remains true even with AI, but AI reduces the drafting and comparison burden significantly.

## Final Principle

The matrix represents real architecture and governance labor.

This layer exists so that labor remains visible rather than being mistaken for automatic structure.
