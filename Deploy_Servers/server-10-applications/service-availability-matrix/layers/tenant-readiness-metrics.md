# Tenant Readiness Metrics

## Purpose

This layer defines the major readiness dimensions the platform should watch across tenants.

It exists so readiness can be discussed in stable categories rather than as one vague feeling.

This is the readiness-metrics layer of the Service Availability Matrix.

## Why This Exists

The platform already uses readiness language such as:
- baseline ready
- parity-ready
- registration-ready

But it is helpful to define which dimensions create that readiness.

## Core Principle

Tenant readiness is multi-dimensional.

A tenant may be strong in one dimension and weaker in another.

## Suggested Readiness Metrics

### 1. Identity Readiness
Measures whether the tenant is recognizable as a business/application line.

Signals include:
- tenant README
- business README

### 2. Business Structure Readiness
Measures whether the tenant has core business anchors.

Signals include:
- business-model README
- canvas overview
- communications README
- strategy README

### 3. Communication Readiness
Measures whether the tenant can explain itself at more than one depth.

Signals include:
- short-form pitch
- deeper audience-facing communication

### 4. Matrix Readiness
Measures whether the tenant is visible to governance comparison.

Signals include:
- tenant matrix entry
- readiness visibility
- profile fit visibility

### 5. Surface Readiness
Measures whether the tenant’s visible business/service surfaces are known.

Signals include:
- website or UI clarity
- role-surface or stakeholder/service-module clarity
- explicit known service areas

### 6. Comparative Readiness
Measures whether the tenant is mature enough to be fairly compared against other tenants.

Signals include:
- baseline satisfied
- profile fit not unclear
- absence is explainable

## Use of Metrics

These metrics can help the platform discuss:
- where a tenant is strong
- where a tenant is still shallow
- whether a tenant is ready for onboarding reference use
- which readiness area needs the next strengthening effort

## Final Principle

Readiness becomes clearer when it has dimensions.

This layer exists so tenant readiness can be evaluated more consistently.
