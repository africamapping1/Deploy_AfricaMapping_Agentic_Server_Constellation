# Manual Review Exit Criteria

## Purpose

This layer defines when a manual Service Availability Matrix review can be considered complete for a given tenant or change event.

It exists so reviews do not remain vague or unfinished.

This is the manual-review-exit layer of the Service Availability Matrix.

## Why This Exists

A manual review is only useful if it reaches a clear stopping point.

Without exit criteria, a review may:

- wander
- miss key questions
- stop too early
- continue too long without added value

This layer provides a clean stopping rule.

## Core Principle

A manual review is complete when the tenant’s baseline, profile fit, parity meaning, and next actions are clear enough for current governance needs.

## Exit Criteria

A manual review may be considered complete when all of the following are true:

### 1. Baseline Clarity
It is clear whether the tenant:
- satisfies the baseline
- partially satisfies the baseline
- fails the baseline

### 2. Communication Minimum Clarity
It is clear whether the tenant has:
- one short-form pitch
- one deeper audience-facing communication

### 3. Matrix Visibility Clarity
It is clear whether:
- the tenant matrix entry exists
- readiness is visible
- profile fit is visible enough

### 4. Surface Clarity
It is clear what the tenant’s major surface pattern is, even if some areas are still evolving.

### 5. Profile Fit Clarity
It is clear whether the tenant:
- strongly fits a profile
- partially fits a profile
- remains unclear

### 6. Parity Meaning Clarity
It is clear whether observed differences are:
- baseline drift
- profile-based
- rollout-based
- exception-based
- still under observation

### 7. Next Action Clarity
It is clear whether the platform should:
- take no action
- note an advisory next step
- correct a real gap
- review profile fit later
- observe further

## What Review Completion Does Not Require

A review does not require:
- perfect certainty
- complete automation
- full bundle stabilization
- advanced maturity in every area

It only requires enough clarity for the current stage.

## Final Principle

A review should end when the platform has enough clarity to trust its next step.

This layer exists so manual review becomes disciplined and repeatable.
