# Manual Effort Estimate

## Purpose

This layer gives a rough estimate of the manual effort required to reach the current Service Availability Matrix state without AI assistance.

It exists so the platform can understand the scale of work represented by the current matrix.

This is the manual-effort layer of the Service Availability Matrix.

## Why This Exists

The matrix now includes:

- many conceptual layers
- many tenant summaries
- multiple registration profiles
- multiple review and governance tools
- parity, promotion, readiness, and rollout logic

This would require substantial effort if done fully by hand.

## Rough Manual Effort Estimate

### Single Strong Generalist
If one strong analyst/writer/operator were doing this manually, a rough estimate to reach the current state would be:

- 60 to 120 focused hours if they already understand the platform well
- 120 to 200+ hours if they are still learning the platform and refining terminology while building

## Why the Range Is Wide

The range depends on:

- how clearly the architecture is already understood
- how many rewrites are needed
- how much coordination is required
- how disciplined the naming and summary system is
- how much of the work is comparison versus original structuring

## Team-Based Manual Equivalent

A small manual team might distribute the work like this:

- analyst / architect defines the model
- technical writer structures and documents it
- product / operations reviewer validates applicability

In that case, total hours may still land in a similar broad range, but spread across multiple people.

## Important Note

This estimate is not a promise or a measured audit.

It is a realistic planning estimate for work of this kind.

## Final Principle

The current matrix represents meaningful labor, not just text.

This layer exists so the human effort equivalent stays visible.
