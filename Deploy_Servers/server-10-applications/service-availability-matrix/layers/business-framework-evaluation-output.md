# Business Framework Evaluation Output

## Purpose

This layer defines the kinds of outputs that the Service Availability Matrix may produce when evaluating the business framework.

It exists so evaluation results can remain understandable and reusable.

This is the evaluation-output layer for business-framework inside the Service Availability Matrix.

## Why This Exists

Evaluation is only useful if its outputs can be read and acted upon.

The platform needs a compact set of possible output forms.

## Typical Output Types

### 1. Presence Output
Examples:
- missing
- partial
- present
- strong

Used for:
- minimum-check style judgments

### 2. Completeness Output
Examples:
- low completeness
- moderate completeness
- high completeness

Used for:
- broad framework coverage judgments

### 3. Readiness Output
Examples:
- early
- emerging
- ready
- mature

Used for:
- forward-use judgments

### 4. Trust Output
Examples:
- low trust
- moderate trust
- high trust

Used for:
- summary and comparison-support judgments

### 5. Promotion Output
Examples:
- no clear signal
- emerging signal
- strong signal

Used for:
- repeated-pattern review

### 6. Review Recommendation Output
Examples:
- continue framework strengthening
- begin SAM comparison use
- prepare governance review
- defer stronger control until more stable

Used for:
- next-step discipline

## Important Note

The matrix does not need to produce all output types every time.

It should produce only the outputs that match the scope of the evaluation being performed.

## Final Principle

Evaluation becomes useful when its outputs are clear enough to guide next action.

This layer exists so business-framework evaluation can speak in disciplined ways.
