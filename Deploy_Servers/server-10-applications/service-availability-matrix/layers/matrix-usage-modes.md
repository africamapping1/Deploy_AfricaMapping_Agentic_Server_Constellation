# Matrix Usage Modes

## Purpose

This layer defines the different ways the Service Availability Matrix can be used by the platform at different maturity stages.

It exists so the matrix is not treated as if it only has one role.

This is the usage-mode layer of the Service Availability Matrix.

## Why This Exists

The matrix can act as:
- documentation
- governance memory
- review discipline
- onboarding guide
- provisioning influence
- early automation source
- future warning/gating source

These are different modes.

Naming them helps the platform stay clear.

## Core Principle

The matrix can be useful before it is fully automated.

Different usage modes are valid at different times.

## Suggested Usage Modes

### 1. Reference Mode
The matrix is used mainly to understand structure and meaning.

### 2. Comparison Mode
The matrix is used to compare tenants, profiles, bundles, and readiness states.

### 3. Review Mode
The matrix is used during manual review and change assessment.

### 4. Onboarding Mode
The matrix is used to guide how new tenants are shaped.

### 5. Governance Mode
The matrix is used to support promotion, standardization, parity, and escalation discussion.

### 6. Advisory Automation Mode
The matrix begins producing simple low-risk signals and summaries.

### 7. Control Mode
The matrix begins influencing warnings, gates, or stronger provisioning logic.

## Current Platform State

The platform is currently strongest in:
- reference mode
- comparison mode
- review mode
- governance mode

It is emerging in:
- onboarding mode

It is earlier in:
- advisory automation mode
- control mode

## Final Principle

The matrix does not need to wait for full automation to be useful.

This layer exists so the platform can use the matrix appropriately at each maturity stage.
