# Tenant Growth Paths

## Purpose

This layer defines the likely directions in which tenants may grow after they become registration-ready.

It exists so the matrix can guide strengthening beyond baseline completion.

This is the growth-path layer of the Service Availability Matrix.

## Why This Exists

A tenant that is registration-ready is not finished.

The platform should still be able to ask:
- what is the next likely maturity direction
- what kind of growth path is this tenant taking
- what should be strengthened next

## Core Principle

Different tenants may grow in different directions after baseline readiness.

Growth should be understood, not assumed.

## Common Growth Paths

### 1. Business-Model Deepening Path
The tenant becomes stronger in:
- value proposition
- segments
- channels
- relationships
- revenue/funding logic
- key resources
- key activities
- key partners
- cost structure

### 2. Communication Deepening Path
The tenant becomes stronger in:
- positioning
- operator explanation
- objection handling
- partner/customer/investor variations

### 3. Surface Expansion Path
The tenant becomes stronger in:
- UI
- dashboards
- customer/employee/stakeholder surfaces
- explicit service modules

### 4. Bundle Stabilization Path
The tenant becomes stronger in:
- clearer bundle identity
- stronger profile alignment
- easier reuse across similar tenants

### 5. Standard Influence Path
The tenant becomes more useful as:
- a reusable reference
- a promotion source
- a baseline teacher for future onboarding

## Current Likely Paths

### `realestate-ai`
Likely current strongest paths:
- business-model deepening
- communication deepening
- role-surface bundle stabilization

### `africamapping`
Likely current strongest paths:
- business-model deepening
- communication deepening
- stakeholder/service-module bundle stabilization

## Final Principle

A tenant should not only be judged by what it has now.

It should also be understood by where it is likely going.

This layer exists so tenant growth stays visible and guideable.
