# Minimum Review Questions

## Purpose

This layer defines the minimum questions that should be asked whenever a tenant, bundle, or profile is reviewed through the Service Availability Matrix.

It exists so review quality stays consistent even before automation exists.

This is the minimum-review-questions layer of the Service Availability Matrix.

## Why This Exists

A matrix can become large.

Without a minimum review discipline, different reviewers may focus on different things and miss important issues.

This layer provides a stable review floor.

## Core Principle

Every meaningful matrix review should answer the same small set of grounding questions before moving into deeper interpretation.

## Tenant Review Questions

For any tenant, ask:

### Identity
- does the tenant have a root README
- does the tenant have a business README

### Baseline
- does the tenant meet the current business minimum baseline
- what baseline item is missing, if any

### Communications
- does the tenant have one short-form pitch
- does the tenant have one deeper audience-facing communication

### Business Structure
- does the tenant have business-model, canvas, and strategy anchors

### Matrix Visibility
- does the tenant have a tenant matrix file
- is readiness visible
- is profile fit visible

### Surface Clarity
- are the main surfaces known
- is absence explained where needed

### Profile Fit
- does the tenant clearly fit a profile
- is the current profile still the best fit

### Parity Meaning
- is the tenant aligned on shared baseline
- are differences explainable by profile or rollout
- is there likely drift

## Bundle Review Questions

For any bundle, ask:

- is this bundle real or only conceptual
- in how many tenants does it appear
- is it comparable across tenants
- is it reusable yet
- is it becoming standard
- should it influence onboarding

## Promotion Review Questions

For any promotion candidate, ask:

- is the pattern repeated
- is the pattern mature
- is the pattern stable
- does promoting it improve future onboarding or parity
- does it belong in tenant space or shared structure

## Rollout Review Questions

For any rollout-sensitive capability, ask:

- is this active, partial, pilot, or planned
- is absence a real problem or rollout timing
- should parity tighten yet

## Final Principle

A good review does not need to start from zero each time.

This layer exists so every matrix review begins from a consistent minimum discipline.
