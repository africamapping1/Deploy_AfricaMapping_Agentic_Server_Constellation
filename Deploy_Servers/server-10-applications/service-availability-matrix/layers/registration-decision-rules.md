# Registration Decision Rules

## Purpose

This layer defines how the platform should decide whether a tenant is ready to be treated as a properly registered business instance.

It exists so registration readiness is not left to vague intuition.

This is the registration-decision layer of the Service Availability Matrix.

## Why This Exists

The matrix already has readiness logic, readiness summaries, and registration baseline thinking.

What is still useful is a rule layer that states more directly:

- when is the answer yes
- when is the answer not yet
- when is the answer yes, but with notes

## Core Principle

Registration should follow visible structural sufficiency.

A tenant should be treated as registered when it has enough baseline structure, communication clarity, and matrix visibility to be governable.

## Decision Questions

A registration decision should ask:

### Identity Sufficiency
- does the tenant have a root README
- does it have a business README

### Business Sufficiency
- does it have business-model, canvas, communications, and strategy anchors

### Communication Sufficiency
- does it have one short-form pitch
- does it have one deeper audience-facing communication

### Matrix Sufficiency
- does it have a tenant matrix entry
- is its readiness visible
- is its profile fit at least identifiable

### Surface Sufficiency
- are its main surfaces at least known or explicitly not applicable

## Suggested Decision Outcomes

### Not Ready
Use when the tenant lacks major baseline anchors and cannot yet be treated as a governed business instance.

### Partially Ready
Use when the tenant has some baseline structure, but important required anchors are still missing.

### Registration-Ready
Use when the tenant satisfies the current baseline and has enough visible shape to enter governed comparison and onboarding logic.

### Registration-Ready With Notes
Use when the tenant is ready, but still has important non-blocking gaps worth tracking.

## Good Current Indicators for “Registration-Ready”

At the current stage, the strongest indicators are:

- shared business minimum baseline satisfied
- one short-form pitch present
- one deeper audience-facing communication present
- tenant matrix file present
- profile fit visible enough for reasonable interpretation

## Relationship to Readiness Gates

Registration decision rules should be consistent with readiness gates.

The decision is the judgment.

The gates are the staged path that leads to it.

## Final Principle

A tenant should be registered because it crossed visible thresholds, not because it merely exists.

This layer exists so registration decisions remain explainable and consistent.
