# Tenant Matrix Summary

## Purpose

This file provides a cross-tenant summary view of the Service Availability Matrix.

It is the first comparative layer above individual tenant files.

Its job is to help the platform quickly see:

- where tenants share a common baseline
- where tenants differ
- where maturity is uneven
- where a reusable standard is emerging
- where future parity checks should focus

## Why This Exists

Individual tenant files are useful for deep inspection.

But the platform also needs a higher-level comparative view.

Without this summary, tenant matrix files remain isolated.

With this summary, the platform can start asking:

- what is common now
- what is uneven now
- what is becoming standard
- what needs attention next

## Current Compared Tenants

At this time, the matrix summary compares:

- `africamapping`
