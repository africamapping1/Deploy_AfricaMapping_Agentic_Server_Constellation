# Tenant Standard Candidates

## Purpose

This file identifies candidate standards derived from cross-tenant comparison.

It exists so repeated patterns and shared gaps across tenants can be converted into reusable platform standards.

---

## Source of Insight

This file is based on comparison between:

- africamapping
- realestate-ai

---

## Shared Structural Gaps

The following gaps are consistently present across tenants:

### 1. Business Model Detail Files

Missing:
- customer/stakeholder segments
- value proposition
- channels
- relationships
- revenue model
- key resources
- key activities
- key partners
- cost structure

### 2. Positioning

Missing:
- clear positioning statements
- competitive or contextual differentiation

### 3. Operator Explanation

Missing:
- how the system is operated internally
- how roles interact with the system

### 4. Objection Handling

Missing:
- common objections
- structured responses

---

## Shared Strength Patterns

The following patterns are consistently strong:

### 1. Business README Structure
- clear and reusable baseline structure

### 2. Canvas Overview
- strong summary of business surface

### 3. Pitch Layers
- elevator pitch
- stakeholder or investor pitch

### 4. Tenant Readability
- both tenants are understandable as business surfaces

---

## Candidate Standards

Based on shared patterns, the following are strong candidates for standardization:

### 1. Minimum Business Model Package
A required set of business-model component files.

### 2. Communication Baseline Package
A required set of communication layers:
- elevator pitch
- stakeholder or investor pitch
- positioning
- operator explanation
- objection handling

### 3. Role-Surface Definition Pattern
Standard way to define:
- customer surface
- employee surface
- interaction points

### 4. Business Summary Structure
Standard way to summarize:
- purpose
- value
- stakeholders
- surfaces

---

## Priority Order

1. Business model detail files
2. Positioning
3. Operator explanation
4. Objection handling

---

## Final Principle

Repeated gaps reveal missing standards.

Repeated strengths reveal reusable patterns.

This file exists so the platform can convert observation into structure.
