# Governance Decision Shortlist

## Purpose

This file lists the next strongest matrix-related decisions that governance will likely need to make as the Service Availability Matrix matures.

It exists so future governance attention can stay focused.

This is the governance-decision shortlist of the tenant matrix.

## Why This Exists

The matrix is now rich enough that some upcoming questions are becoming visible.

Instead of waiting for them to become urgent, the platform should name them early.

## Likely Upcoming Decisions

### 1. Confirm Shared Tenant Business Minimum Baseline

Question:
- should the current baseline now be treated as a registration-default standard

Why this matters:
- this is the strongest repeated pattern across tenants

## 2. Confirm Short-Form + Deeper Communication Baseline

Question:
- should every tenant be expected to have one short-form pitch and one deeper audience-facing communication

Why this matters:
- this pattern is now strong across current tenants

## 3. Formalize Role-Surface Tenant-Type Expectations

Question:
- what should a role-surface tenant be expected to have by default

Why this matters:
- `realestate-ai` is already a strong example

## 4. Formalize Stakeholder / Service-Module Tenant-Type Expectations

Question:
- what should a stakeholder/service-module tenant be expected to have by default

Why this matters:
- `africamapping` is already a strong example

## 5. Decide Earliest Automation Scope

Question:
- which baseline checks should move into simple automation first

Why this matters:
- the matrix now has enough stable structure for careful early automation

## 6. Decide Earliest Doctor Integration Scope

Question:
- which matrix signals should be allowed to appear in doctor-style visibility first

Why this matters:
- matrix health visibility is one of the clearest next operational bridges

## 7. Decide When Baseline Failures Become Warning-Level

Question:
- at what maturity point should missing baseline items move from documentary weakness to stronger review concern

Why this matters:
- enforcement should grow gradually, but intentionally

## Final Principle

A platform matures when it can name its next decisions before they become emergencies.

This file exists so matrix-related governance work can stay deliberate.
