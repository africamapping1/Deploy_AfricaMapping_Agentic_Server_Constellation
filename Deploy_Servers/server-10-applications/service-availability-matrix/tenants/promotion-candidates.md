# Tenant Promotion Candidates

## Purpose

This file provides a comparative summary of the strongest current promotion candidates emerging from tenant structures, tenant business layers, and observed bundle patterns.

It exists so the platform can explicitly see which patterns may now be ready to move upward into:

- shared business framework
- application-plane standard structure
- capability bundle logic
- future registration defaults

This is the tenant-derived promotion candidate layer of the Service Availability Matrix.

## Why This Exists

The platform is already learning from repeated tenant patterns.

But those patterns should not remain hidden inside separate tenant files.

This file exists to collect the strongest promotion candidates in one place and make them visible for governance, parity, and future onboarding design.

## Current Compared Tenants

At this time, the candidate summary compares:

- `africamapping`
- `realestate-ai`

Future tenants should be added here as they are created.

## Promotion Candidates That Are Still Too Early

The platform should recognize that some patterns are still too weak or too uneven for stronger promotion.

Examples include:

- integration-oriented bundle as a universal standard
- detailed business-model component files
- positioning as universal default
- objection handling as universal default
- operator explanation as universal default

These are visible next-step candidates, but not yet strong enough for strong upward promotion.

## Suggested Promotion Priorities

### Highest Priority
- tenant business minimum baseline
- short-form pitch baseline
- deeper audience-facing pitch expectation

### Medium Priority
- role-surface tenant-type bundle
- stakeholder/service-module tenant-type bundle

### Lower Priority / Continue Observation
- integration-oriented bundle
- advanced communication package defaults
- detailed business-model file set as registration default

## Governance Recommendation

Promotion should now begin cautiously from the strongest shared evidence.

That means the platform can begin to treat the following as increasingly stable:

- tenant business minimum baseline
- one short-form pitch
- one deeper audience-facing communication
- matrix visibility for every tenant

These are the strongest promotion candidates currently supported by observed tenant structure.

## Final Interpretation

The platform is no longer only documenting tenants.

It is beginning to extract reusable architecture from them.

This file exists so the strongest promotion candidates remain visible, discussable, and governable.
