# Tenant Control Target

## Purpose

This layer defines tenant application spaces as formal control targets of the Service Availability Matrix.

It exists so the platform can evaluate real tenant business/application surfaces rather than only abstract framework definitions.

This is the tenant control-target layer of the Service Availability Matrix.

## Why This Exists

Tenant spaces in `server-10-applications/apps/` represent:

- real business instances
- real application surfaces
- real usage contexts

They are where the business framework becomes applied reality.

## What This Means

The matrix should be able to evaluate tenants for:

- business definition presence
- completeness of business expression
- readiness for governance use
- comparability across tenants
- stability of structure

## Important Note

Tenants are not framework abstractions.

They are concrete applications of the framework.

## Final Principle

A matrix becomes meaningful when it evaluates real systems, not only theory.

This layer exists so tenant evaluation becomes explicit.
