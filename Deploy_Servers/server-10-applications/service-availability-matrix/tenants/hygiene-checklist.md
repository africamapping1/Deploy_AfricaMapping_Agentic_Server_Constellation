# Matrix Hygiene Checklist

## Purpose

This file provides a simple checklist for keeping the Service Availability Matrix clean and trustworthy.

It exists so matrix quality does not degrade as more files are added.

This is the matrix-hygiene layer of the tenant matrix.

## Why This Exists

A large matrix can weaken through small maintenance problems such as:

- typos
- stale summaries
- missing references
- duplicate meanings
- outdated judgments

This checklist helps prevent that.

## Hygiene Checklist

### 1. File Name Check
Confirm:
- file names use `.md`
- naming is consistent
- no obvious typos exist

### 2. Summary Sync Check
Confirm:
- summaries still reflect current logic
- major tenant changes are represented
- promotion and readiness summaries remain current

### 3. Tenant Coverage Check
Confirm:
- each modeled tenant has a tenant matrix file
- each tenant is visible in expected summary layers

### 4. Broken Logic Check
Confirm:
- no layer contradicts another layer obviously
- no summary still reflects outdated assumptions
- no profile statement is clearly stale

### 5. Redundancy Check
Confirm:
- new files are adding value
- meanings are not duplicated without reason
- summary hierarchy remains clear

### 6. Current Action Check
Confirm:
- next priorities still make sense
- operational gap summaries still reflect reality
- phase completion summary still feels honest

## Suggested Use

Use this checklist:
- after large batch additions
- before commit checkpoints
- before trying to automate anything
- before treating the matrix as stronger operational truth

## Final Principle

Matrix quality depends on hygiene as much as theory.

This file exists so the matrix remains trustworthy as it grows.
