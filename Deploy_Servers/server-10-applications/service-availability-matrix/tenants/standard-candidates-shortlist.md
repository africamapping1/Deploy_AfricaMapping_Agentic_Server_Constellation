# Standard Candidates Shortlist

## Purpose

This file provides a short prioritized list of the tenant-derived patterns that are currently closest to becoming stronger standards.

It exists so the platform can focus attention on the most realistic next standards rather than treating every candidate equally.

This is the shortlist layer above the broader promotion and standardization summaries.

## Why This Exists

The matrix already contains many observations, candidates, and summaries.

A shortlist is useful because it answers a more focused question:

What is most ready next?

Without a shortlist, the platform may spread attention too widely.

With a shortlist, the platform can move more deliberately.

## Current Shortlist

### 1. Tenant Business Minimum Baseline

#### Why It Is Here
This is the strongest repeated structure across current tenants.

#### Evidence
Both `africamapping` and `realestate-ai` currently support:

- tenant README
- business README
- business-model README
- canvas overview
- communications README
- strategy README
- short-form pitch
- deeper audience-facing communication

#### Current Judgment
- strongest near-term standard

## 2. Communications Minimum Baseline

- deeper audience-facing communication

#### Current Judgment

## 3. Matrix Enrollment Baseline

#### Current Judgment

#### Why It Is Here

#### Evidence
`africamapping` shows the strongest current stakeholder/service-module pattern.

#### Current Judgment
- strong tenant-type standard candidate
- not yet universal

## Items Not Yet on the Shortlist

The following are still too early or too uneven to be on the strongest shortlist:

- integration-oriented bundle as standard
- hybrid profile as standard onboarding profile
- detailed business-model component defaults
- advanced communications defaults such as objection handling or positioning

These remain important, but not highest-confidence next standards.

## Why This Matters

This shortlist helps the platform decide what to harden next without losing flexibility.

It supports:
- better review focus
- clearer governance discussion
- better onboarding evolution
- stronger use of evidence

## Final Principle

A mature platform should know not only what could become standard.

It should know what is closest now.

This file exists so standardization effort becomes more focused and disciplined.
