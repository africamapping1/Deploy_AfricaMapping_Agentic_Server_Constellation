# Tenant Evaluation Scope

## Purpose

This layer defines what parts of tenant application spaces fall within Service Availability Matrix evaluation scope.

It exists so the platform evaluates the right tenant surfaces.

## In-Scope Areas

- business definition within tenant
- application structure
- business-framework alignment
- tenant summaries
- readiness for use

## Light Scope

- experimental features
- early drafts
- evolving modules

## Out of Scope

- raw notes
- incomplete placeholders
- unrelated assets

## Final Principle

Evaluate what is stable enough to judge.
