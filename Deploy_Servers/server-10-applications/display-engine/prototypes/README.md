# Prototypes

## Purpose

This folder holds early working display examples that help evolve the display engine into reusable form.

It exists so the platform can experiment with real display behavior without confusing prototypes with stable reusable engine components.

## Role

Prototypes are useful for:

- testing ideas quickly
- proving display concepts
- validating map behavior
- identifying what should later move into reusable models, views, controllers, services, and widgets

## Current Direction

The existing first map viewer should be treated as an early prototype.

Its role is to prove that the system can:

- load a basemap
- load a GeoJSON layer
- show geospatial output in the browser

## Rule

A prototype is allowed to be simple and direct.

But once a pattern proves useful, it should be moved into reusable engine structure.

## Final Principle

Prototypes prove ideas.

The engine preserves and reuses the good ones.
