# Governance

## Purpose

This folder contains the business-framework governance surfaces.

It exists so the platform can define the rules by which business surfaces are reviewed, compared, judged as complete, and later made readable to the Service Availability Matrix.

## Typical Contents

This folder may include files such as:

- minimum business definition
- completeness rules
- readiness rules
- comparison basis
- profile expectations
- summary discipline
- promotion candidate logic
- review rhythm
- SAM socket notes

## Final Principle

Governance is for disciplined business interpretation, not tenant execution.
