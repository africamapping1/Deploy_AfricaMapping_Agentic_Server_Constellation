# Narrator Positioning

## Role

The narrator is the constellation's explanation layer.

It does not create truth.
It interprets truth already present in:

- architecture
- flows
- state
- policies
- deployment history

## Position in the System

The narrator serves three contexts:

- internal (engineering and governance)
- operator (deployment and runtime use)
- sales (product explanation and value communication)

## Principle

The narrator communicates the system’s intent, state, and value across engineering, operations, and product contexts.
