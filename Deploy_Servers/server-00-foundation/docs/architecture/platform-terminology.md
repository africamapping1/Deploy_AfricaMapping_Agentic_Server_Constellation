# Platform Terminology

## Purpose

This document defines the official language of the constellation.

It ensures that Narrator, Strategist, Governor, and all related platform components use clear, consistent, and governed terminology.

## Principle

Clear language produces clear reasoning.
Clear reasoning produces safe governance.

---

## Internal and External Language

Some terms are used internally for system identity.
Some terms are used externally for industry clarity.

Both must remain aligned.

---

## Core Terms

### Deploy Group
**Internal Name:** Deploy Group  
**External / Industry Name:** Platform Steward  
**Definition:** The founding stewardship layer that establishes, deploys, and guides the constellation.

### Linode
**Internal Name:** Linode  
**External / Industry Name:** Infrastructure Provider / IaaS Layer  
**Definition:** The raw infrastructure hosting environment on which the constellation runs.

### Agentic Server Constellation
**Internal Name:** Constellation  
**External / Industry Name:** Governed Platform / PaaS-like Operational Platform  
**Definition:** The living, governed platform composed of multiple coordinated servers, flows, and governance processes.

### Narrator
**Internal Name:** Narrator  
**External / Industry Name:** Explanation Layer / Operational Narration Layer  
**Definition:** The layer that explains, records, validates, compares, and preserves system truth.

### Strategist
**Internal Name:** Strategist  
**External / Industry Name:** Strategy Layer / Decision Support Layer  
**Definition:** The layer that interprets narrator outputs and provides directional guidance.

### Governor
**Internal Name:** Governor  
**External / Industry Name:** Policy Enforcement Layer  
**Definition:** The layer that evaluates strategist outputs and enforces decisions according to policy.

### Governance Loop
**Internal Name:** Governance Loop  
**External / Industry Name:** Governance Loop  
**Definition:** The control cycle formed by Narrator, Strategist, and Governor.

### Server-10 Applications
**Internal Name:** Application Layer  
**External / Industry Name:** SaaS-like Application Layer  
**Definition:** The layer of business-facing applications that consume infrastructure without modifying platform internals.

### AfricaMapping
**Internal Name:** AfricaMapping  
**External / Industry Name:** Business Consumer / First Tenant  
**Definition:** The first business entity consuming the platform and application layer.

---

## Layer Language

### Infrastructure Layer
Provides:
- stability
- governed execution
- observability
- runtime memory
- controlled evolution

### Application Layer
Provides:
- business-facing workflows
- interfaces
- modular capabilities
- operational tools

### Business Layer
Provides:
- customer value
- market-facing outcomes
- project and program execution
- organizational operation

---

## Language Rules for Narrator

Narrator must:

- prefer defined terms over improvised terms
- remain consistent across summaries and reports
- distinguish infrastructure from applications
- distinguish applications from business
- distinguish internal names from external names when needed

Narrator must not:

- invent unstable synonyms
- blur platform and application boundaries
- confuse business consumers with infrastructure operators

---

## Governance Rule

When terminology changes, platform documents, Narrator outputs, Strategist outputs, and Governor policies must remain aligned.

## Closing Principle

Language is part of governance.

The constellation must describe itself clearly in order to evolve safely.
