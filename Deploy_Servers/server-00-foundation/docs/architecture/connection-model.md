# Connection Model

## Core Idea

The invisible line that ties the constellation together is shared governed state across participating servers.

This connection is not merely network reachability.
It is the combination of:

- shared doctrine
- shared lifecycle
- shared deployment state
- shared observability
- shared participation in flows

## Sources of Connection

### Foundation
`server-00-foundation` defines the common language:
- lifecycle
- standards
- schemas
- flow doctrine
- architecture meaning

### Deployment
`server-05-ci-cd` writes the remote state that records what is present, what changed, and what condition the constellation is in.

### Monitoring
`server-06-monitoring` reads and interprets that state to determine whether the constellation is steady, degraded, failed, or evolving.

## Connection Principle

A server is not meaningfully connected merely because it exists on a network.

It is connected when it:
- is recognized by the constellation
- participates in governed flows
- contributes observable state
- can be evaluated as part of the whole

## Operational Meaning

The invisible line becomes real when deployment writes shared state and monitoring can read it back as evidence of coordinated life.
